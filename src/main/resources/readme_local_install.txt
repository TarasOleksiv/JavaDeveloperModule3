Структура папки "resources".
    1) config.properties - файл конфігурації проекту:
        Файл містить параметри шляхів до скриптів ініціалізації та наповнення бази.
    2) DBscheme.png - схема бази.
    3) hibernate.cfg.xml - файл конфігурації hibernate.
        ПЕРЕД ЗАПУСКОМ ПРОГРАМИ НЕОБХІДНО ПРОПИСАТИ В НЬОМУ ПРАВИЛЬНІ ЗНАЧЕННЯ ДЛЯ ЮЗЕРА ТА ПАРОЛЮ НА СВІЙ СЕРВЕР!
    4) initDB.sql - створює всі необхідні таблиці на базі.
    5) populateDB.sql - заповнює даними таблиці.
    6) readme_local_install.txt - файл з інструкцією.

Схема запуску програми на локальній машині.
    1) ПЕРЕД ЗАПУСКОМ ПРОГРАМИ НЕОБХІДНО ПРОПИСАТИ у файлі resources\hibernate.cfg.xml ПРАВИЛЬНІ ЗНАЧЕННЯ ДЛЯ ЮЗЕРА ТА ПАРОЛЮ НА СВІЙ СЕРВЕР!
    2) Запустити програму.
        При запуску програма перевіряє чи на серевері mysql існує база, прописана у файлі hibernate.cfg.xml з пункту 1;
            (для перевірки використовується jdbc connection, який зчитує параметри з'єднання з hibernate configuration).
            Якщо база не існує, то програма автоматично запускає на виконання 2 скрипти з папки resources (initDB.sql, populateDB.sql):
                ці скрипти створюють на сервері базу, таблиці та заливають дані в таблиці;
                (для автоматичного запуску скриптів використовується бібліотека ibatis-sqlmap, яка в свою чергу використовує
                jdbc connection, котрий зчитує параметри з'єднання з hibernate configuration).
            Якщо база існує, то програма продовжує роботу.
            Надалі використовується лише hibernate.
    3) Продовжити роботи в Головному меню.
